このアクショングループは、製品要件の分析、ユーザーストーリーの作成、競合分析、および製品要件書(PRD)の作成を行うプロダクトマネージャーエージェントの機能を提供します。

以下の関数を含みます：

1. analyze_requirement
   説明: ユーザーの要件を分析し、主要な機能、ターゲットユーザー、潜在的な課題を抽出します。
   パラメータ:
   - requirement: ユーザーから提供された要件の詳細説明。例: "モバイルアプリで家計簿を管理したい"
     タイプ: string
     必須: はい
   - project_id: プロジェクトを識別するための一意のID。指定しない場合は自動生成されます。
     タイプ: string
     必須: いいえ
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ

2. create_user_stories
   説明: 要件に基づいてユーザーストーリーを作成します。各ストーリーは「〜として、〜したい、なぜなら〜」の形式で表現されます。
   パラメータ:
   - requirement: ユーザーから提供された要件の詳細説明。
     タイプ: string
     必須: はい
   - analysis_id: 以前の要件分析のID。
     タイプ: string
     必須: いいえ
   - project_id: プロジェクトを識別するための一意のID。
     タイプ: string
     必須: いいえ
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ

3. create_competitive_analysis
   説明: 要件に基づいて競合分析を行い、主要な競合他社、その強みと弱み、市場での位置づけを特定します。
   パラメータ:
   - requirement: ユーザーから提供された要件の詳細説明。
     タイプ: string
     必須: はい
   - project_id: プロジェクトを識別するための一意のID。
     タイプ: string
     必須: いいえ
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ

4. create_product_requirement_doc
   説明: ユーザーストーリーと競合分析に基づいて、製品要件書(PRD)を作成します。概要、ユーザーストーリー、機能、非機能要件、タイムラインなどのセクションが含まれます。
   パラメータ:
   - requirement: ユーザーから提供された要件の詳細説明。
     タイプ: string
     必須: はい
   - stories_id: 以前に作成されたユーザーストーリーのID。
     タイプ: string
     必須: いいえ
   - competitive_analysis_id: 以前に作成された競合分析のID。
     タイプ: string
     必須: いいえ
   - project_id: プロジェクトを識別するための一意のID。
     タイプ: string
     必須: いいえ
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ

このアクショングループは、コードの実装、コードレビュー、バグ修正を行うエンジニアエージェントの機能を提供します。

以下の関数を含みます：

1. implement_code
   説明: 要件とアーキテクチャ設計に基づいてコードを実装します。適切な構造化、ドキュメント化、テスト済みのコードを提供します。
   パラメータ:
   - requirement: ユーザーから提供された要件の詳細説明。
     タイプ: string
     必須: はい
   - prd_id: 以前に作成された製品要件書(PRD)のID。
     タイプ: string
     必須: いいえ
   - architecture_id: 以前に作成されたアーキテクチャのID。
     タイプ: string
     必須: いいえ
   - project_id: プロジェクトを識別するための一意のID。指定しない場合は自動生成されます。
     タイプ: string
     必須: いいえ
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ

2. review_code
   説明: 実装されたコードをレビューします。コード品質の評価、バグの特定、改善提案、要件への適合性確認を行います。
   パラメータ:
   - implementation_id: レビューする実装のID。
     タイプ: string
     必須: はい
   - project_id: プロジェクトを識別するための一意のID。
     タイプ: string
     必須: はい
   - requirement: ユーザーから提供された要件の詳細説明。
     タイプ: string
     必須: いいえ
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ

3. fix_bugs
   説明: レビューに基づいてコード内のバグを修正します。問題を分析し、解決策を提供し、コードが要件を満たすことを確認します。
   パラメータ:
   - implementation_id: 修正する実装のID。
     タイプ: string
     必須: はい
   - review_id: 以前に作成されたコードレビューのID。
     タイプ: string
     必須: いいえ
   - project_id: プロジェクトを識別するための一意のID。
     タイプ: string
     必須: はい
   - user_id: リクエストを行ったユーザーのID。
     タイプ: string
     必須: いいえ
